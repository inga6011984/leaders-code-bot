# Leader's Code — инструкция по запуску

Что это: клиент проходит квиз по ссылке, вводит имя, в конце видит только
экран «Готово» — без единой цифры. Полный профиль в это время улетает вам
в Telegram. Ниже — как всё это включить с нуля, шаг за шагом, без
предварительного опыта.

Всего понадобится: аккаунт Telegram (уже есть), аккаунт GitHub (создадим),
аккаунт Vercel (создадим). Всё бесплатно.

---

## Шаг 1. Создать Telegram-бота

1. В Telegram найдите **@BotFather** (это официальный бот Telegram) и
   откройте с ним диалог.
2. Отправьте команду `/newbot`.
3. Придумайте имя бота (например, `Leader's Code Notifier`) и юзернейм
   (должен заканчиваться на `bot`, например `leaderscode_inga_bot`).
4. BotFather пришлёт **токен** вида `123456789:AAExampleTokenHere`.
   Скопируйте и сохраните его — это `TELEGRAM_BOT_TOKEN`.

## Шаг 2. Узнать свой chat_id

1. Найдите в Telegram бота **@userinfobot** и напишите ему любое сообщение —
   он пришлёт в ответ ваш `Id` (набор цифр). Это `TELEGRAM_CHAT_ID`.
2. Затем откройте диалог с **вашим новым ботом** (тем, что создали в Шаге 1)
   и отправьте ему `/start` — это обязательно, иначе бот не сможет писать
   вам первым.

## Шаг 3. Выложить код на GitHub

1. Зайдите на [github.com](https://github.com) и зарегистрируйтесь
   (кнопка Sign up).
2. Нажмите **New repository**, назовите его, например, `leaders-code`,
   оставьте Public или Private — не важно, нажмите **Create repository**.
3. На странице репозитория нажмите **uploading an existing file** (или
   «Add file → Upload files»).
4. Перетащите туда **все файлы и папки из этого проекта** (сохраняя
   структуру: `src/`, `api/`, `package.json` и т.д.) и нажмите
   **Commit changes**.

## Шаг 4. Развернуть на Vercel

1. Зайдите на [vercel.com](https://vercel.com), нажмите **Sign Up** и
   выберите вход через GitHub (Continue with GitHub) — это свяжет аккаунты
   автоматически.
2. На дашборде нажмите **Add New → Project**.
3. Найдите репозиторий `leaders-code` и нажмите **Import**. Vercel сам
   определит, что это Vite-проект — ничего менять не нужно.
4. Перед деплоем разверните раздел **Environment Variables** и добавьте
   две переменные:
   - `TELEGRAM_BOT_TOKEN` — значение из Шага 1
   - `TELEGRAM_CHAT_ID` — значение из Шага 2
5. Нажмите **Deploy**. Через минуту-две Vercel даст вам публичную ссылку
   вида `https://leaders-code-inga.vercel.app` — это и есть ссылка, которую
   вы отправляете клиенту.

## Шаг 5. Проверить

Откройте полученную ссылку сами, введите тестовое имя, пройдите квиз до
конца. В Telegram-чате с ботом должно прийти сообщение с профилем.
Если не пришло — проверьте в Vercel: **Project → Deployments →
(последний деплой) → Functions → api/notify** там будут видны ошибки,
если токен или chat_id указаны неверно.

---

## Как это устроено (коротко)

- `src/App.jsx` — сам квиз. В конце теста он молча отправляет отчёт
  на `/api/notify`, что бы ни случилось с этой отправкой — клиент видит
  только «Готово».
- `api/notify.js` — серверная функция (работает на Vercel, не в браузере).
  Только она знает токен бота, поэтому его нельзя увидеть через код
  страницы или консоль браузера.
- Дашборд коуча (`CoachScreen`) остался в коде, но кнопка перехода к нему
  убрана из клиентского экрана — обычный клиент никогда до него не
  доберётся.

## Заметки на будущее

- Сейчас `/api/notify` принимает запросы от кого угодно (без проверки
  «отправлено именно из вашего квиза»). Для внутреннего инструмента на
  несколько клиентов это не критично, но если ссылку начнут пересылать
  массово — можно добавить простой секретный параметр, дайте знать.
- Tailwind сейчас подключён через CDN-скрипт в `index.html` — это самый
  простой вариант для старта, без дополнительной настройки сборки. Если
  позже понадобится ускорить загрузку страницы, можно перейти на
  полноценную сборку Tailwind — это отдельная, не срочная задача.
- Если захотите несколько разных коучей/ссылок с разными chat_id —
  дайте знать, это добавляется через параметр в ссылке (например,
  `?coach=inga`).
